## 一、产品概述

Seedance 2.0 是字节跳动于 2026 年 2 月发布的新一代 AI 视频生成模型，核心突破在于**多模态参考系统**——不再是单一的"文生视频"，而是可以同时输入图片、视频、音频和文本，像导演一样精确控制视频生成的每一个细节。

**核心定位转变：** 从"提示词猜盲盒" → "精准导演控制台"

---

## 二、技术规格

| 参数 | 规格 |
|------|------|
| 图片输入 | 最多 9 张 |
| 视频输入 | 最多 3 个，总时长 ≤ 15 秒 |
| 音频输入 | 最多 3 个 MP3，总时长 ≤ 15 秒 |
| 文本输入 | 自然语言提示词（支持中英文，可数千字） |
| 输出时长 | 4–15 秒（用户可选） |
| 输出分辨率 | 最高 2K / 1080P |
| 总文件上限 | 每次生成最多 12 个文件 |
| 音频输出 | 原生音效和配乐生成 |
| 口型同步 | 支持 8+ 种语言的音素级唇语同步 |
| 生成速度 | 比上一代快 30% |

## 三、核心机制：@ 参考系统

### 3.1 @ 标签语法

这是 Seedance 2.0 最核心的交互机制。上传素材后，系统自动标记为 `@Image1`、`@Video1`、`@Audio1` 等，在提示词中用 @ 引用即可指定每个素材的用途。

**基础语法示例：**
```
@Image1 作为首帧，参考 @Video1 的镜头运动，使用 @Audio1 作为背景音乐
```

### 3.2 常用参考模式

| 用途 | 提示词模板 |
|------|-----------|
| 设定首帧 | `@Image1 作为第一帧` |
| 设定末帧 | `@Image1 作为首帧，@Image2 作为末帧`（创建过渡） |
| 参考动作 | `参考 @Video1 的打斗编排` |
| 复刻运镜 | `完全复刻 @Video1 的镜头运动和转场` |
| 添加音乐 | `使用 @Audio1 作为背景音乐` |
| 延长视频 | `将 @Video1 延长 5 秒` |
| 替换角色 | `将 @Video1 中的女性替换为 @Image1` |
| 角色锁定 | `@Image3 是主角`（保持全程脸部一致） |

### 3.3 两种创作模式

1. **首/末帧模式（First/Last Frame）：** 简单的图片到视频，一张参考图 + 文本提示词
2. **全方位参考模式（Universal/All-Round Reference）：** 混合多种模态，专业级导演控制

## 四、十大核心能力

### 4.1 基础生成质量提升
- 物理准确性（物体碰撞、流体、重力符合真实世界）
- 流畅运动（动量和时序正确）
- 精确指令跟随
- 视觉风格一致性

### 4.2 多模态参考系统
在单个提示词中组合：视频动作 + 图片角色 + 音频节奏 + 文本叙事

### 4.3 角色/物体一致性
- 面部一致性（解决"变脸"顽疾）
- 产品细节保持（Logo、文字、精细纹理）
- 场景连贯性
- 风格锁定（防止生成过程中视觉漂移）

### 4.4 运镜与动作复刻
上传参考视频后可提取：
- 复杂编排（打斗、舞蹈）
- 镜头技巧（推拉摇移跟、航拍、环绕）
- 剪辑节奏
- 特殊运镜（希区柯克变焦、甩镜头、环绕镜头）

### 4.5 创意模板复刻
复刻整个创意概念：广告格式、视觉特效、电影技巧、剪辑风格

### 4.6 视频延长（接着拍）
基于原视频逻辑进行智能续写，保持叙事连贯

### 4.7 视频编辑
在已有视频上修改：角色替换、元素增删、风格迁移、叙事调整

### 4.8 音画同步生成
- 唇语同步对白（多语言）
- 音效匹配画面动作
- 背景音乐跟随视觉节奏
- 带情感表达的配音

### 4.9 卡点剪辑（Beat-Synced Editing）
上传音乐轨道 + 图片/视频素材，自动生成踩点视频

### 4.10 一镜到底（One-Take Continuity）
生成长时间、无剪切的连续运动镜头

## 五、镜头运动系统（关键差异化技能）

> **核心观点：** 镜头运动是 AI 视频质量中影响最大的单一变量。相同的场景描述 + 不同的镜头指令 = 截然不同的结果。

### 5.1 基础镜头运动（Level 1）

| 运动类型 | 说明 | 适用场景 |
|----------|------|---------|
| Pan（摇镜） | 水平旋转 | 展示宽广场景 |
| Tilt（俯仰） | 垂直旋转 | 展示高度对比 |
| Zoom（变焦） | 镜头拉近/推远 | 突出重点元素 |
| Dolly（推轨） | 轨道推进/后退 | 接近或远离主体 |
| Truck（横移） | 横向平移 | 跟随移动主体 |
| Crane（升降） | 戏剧性升/降 | 宏大揭示 |
| Orbit（环绕） | 围绕主体的圆形运动 | 360度展示 |
| Tracking（跟踪） | 跟随移动物体 | 保持对焦主体 |
| Static（固定） | 固定位置 | 稳定画面 |
| Push / Pull | 缓慢前进/后退 | 渐进式揭示 |

### 5.2 修饰词系统（Level 2）

**速度修饰词：**
- `Slow` — 悬念、怀旧、抒情
- `Fast / Rapid` — 紧张、紧迫
- `Subtle` — 微小运动、增强沉浸感
- `Gradual` — 渐进变化
- `Sudden` — 冲击、反转

**情绪修饰词：**
- `Cinematic` — 专业电影质感
- `Aggressive` — 恐怖、动作、追逐
- `Dreamy` — 梦幻、回忆
- `Intimate` — 亲密、情感细节
- `Epic` — 宏大、壮观
- `Dynamic` — 活力、变化

**风格修饰词：**
- `Handheld` — 纪录片感、真实
- `Aerial` — 鸟瞰、宏大
- `Dutch Angle` — 倾斜、不安
- `Gimbal` — 稳定器、专业平滑
- `POV` — 第一人称、沉浸
- `Steadicam` — 平滑跟拍

### 5.3 组合镜头运动（Level 3）

| 组合 | 效果 | 示例 |
|------|------|------|
| Orbit + Zoom In | 强烈视觉冲击 | 环绕古雕像同时缓慢推近 |
| Crane Up + Pan | 宏大氛围 | 从地面升起同时横摇战场 |
| Dolly Zoom（希区柯克变焦） | 眩晕、心理冲击 | 角色恍然大悟时的变焦 |
| Tracking + Handheld Shake | 极度紧张 | 快速跟踪穿越森林逃亡 |

> **关键规则：** 每次提示词中组合镜头运动不超过 2-3 种，用 "+" 或 "while" 连接。

## 七、实用应用场景

| 场景 | 方法 |
|------|------|
| **广告/电商** | 产品图 + 品牌视频参考 + 品牌音乐 → 产品展示视频 |

| **分镜转视频** | 分镜画稿作参考图 + 运动描述 → 动画序列 |
| **模板化创作** | 找到喜欢的视频风格 → 上传为参考 → 用自己的角色/场景生成相同风格 |
| **短片制作** | 2张照片 → 4段5秒视频 → 组合成30秒电影级短片 |

## 八、最佳实践（8条黄金规则）

1. **明确参考用途** — 写清楚每个文件的用途。`参考 @Video1 的镜头运动` 远优于只提及视频。
2. **优先级分配** — 12个文件限制下，优先选择影响最大的素材（运镜参考视频通常比第4张参考图重要）。
3. **检查 @ 引用** — 多文件时仔细核对每个 @标签 对应的实际文件。
4. **区分编辑 vs 参考** — 明确是"修改该视频"还是"从中提取特质用于新内容"。
5. **时长对齐** — 延长5秒，就设5秒生成时长。不匹配会导致不一致。
6. **组合镜头不超过3种** — 超过会导致指令冲突。
7. **使用精确术语** — 避免模糊的"移动"，改用 `平滑3秒推轨前进` + `稳定器拍摄`。
8. **使用高质量参考素材** — 输入模糊，输出就模糊。尽量用 2K/4K 图片。

## 十一、可直接参考的关键点总结

1. **@ 参考系统是核心竞争力** — 这是目前最直观的多模态控制方式
2. **镜头语言素养是关键** — 大多数用户只描述内容，忽略镜头运动，这是提升质量的最大杠杆
3. **组合素材 > 单一提示词** — 图片定角色 + 视频定运镜 + 音频定节奏 = 专业级输出
4. **一致性问题已基本解决** — 通过图片参考锁定人物形象
5. **原生音频是差异化优势** — 自动生成匹配画面的音效、环境音、口型同步
6. **视频编辑能力扩展了用途** — 不仅能生成，还能在已有视频上做修改
7. **速度优势对工作流很重要** — 30%的速度提升让快速迭代成为可能
8. **模板化创作降低了门槛** — 找到好的参考视频，即可批量复刻风格
